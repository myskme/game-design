# CLAUDE.md —— 第二大脑 vault

本文件为 Claude Code 提供这个 Obsidian 笔记库的上下文。
**复制到 vault 根目录**,Claude 在本地或网页端打开本仓库时会自动读取。

## 这是什么

这是一个 Obsidian 笔记库(vault),作为「第二大脑」使用。
纯 Markdown,无构建步骤。通过 Git 在 iPhone / iPad / 两台 MacBook 间同步。

## 目录约定(PARA 风格,可按需调整)

- `00-inbox/` —— 速记、未整理的内容。新想法先丢这里。
- `10-daily/` —— 每日笔记,文件名 `YYYY-MM-DD.md`。
- `20-notes/` —— 整理后的长期知识笔记(永久笔记)。
- `30-projects/` —— 有明确目标和截止日期的项目。
- `40-areas/` —— 需要长期维护的领域(健康、财务、教学等)。
- `90-archive/` —— 已完成 / 不再活跃的内容。
- `_moc/` —— 内容地图(Map of Content),各主题的索引页。
- `.obsidian/` —— Obsidian 配置,请勿手动改动。

## Claude 在这个库里该怎么做

### 整理 inbox(常用)
当被要求「整理收件箱」时:
1. 逐个读取 `00-inbox/` 下的笔记。
2. 为每条补全 frontmatter:`title`、`created`、`tags`。
3. 判断归属,移动到 `20-notes/` / `30-projects/` / `40-areas/`。
4. 用 `[[双向链接]]` 关联到相关的已有笔记。
5. 内容太碎、无法归类的,保留在 inbox 并说明原因。

### 生成每日摘要
当被要求「写今日摘要」时,在 `10-daily/YYYY-MM-DD.md` 生成:
- 当天新增 / 修改的笔记列表(带链接)。
- 3-5 条关键要点提炼。
- 待跟进事项(从笔记里出现的 TODO / 未决问题提取)。

### 维护 MOC
当某个标签 / 主题的笔记超过约 5 条时,在 `_moc/` 下建或更新该主题的索引页,
用链接把相关笔记组织起来。

## 硬性规则

- **绝不删除笔记**。需要清理时移动到 `90-archive/`,并在提交信息里说明。
- 不修改 `.obsidian/` 下任何文件。
- 改动他人原始记录时保留原文,新增内容用清晰的小标题分隔。
- 每次整理完成后做一次 Git 提交,提交信息用简洁祈使句(中英文均可)。
- 笔记是私人内容,不要把片段贴到外部服务或公开仓库。

## 同步说明

本库由 Obsidian Git 插件(MacBook / 移动端)自动提交推送。
Claude 在做批量改动前,可先 `git pull` 取最新,避免与设备端同步冲突。
