# 一键口令(给 Claude Code,一步到位)

把下面整段粘进 Claude Code,**只需替换开头两个 `< >` 的值**,其余全自动。

> ⚠️ 必须在**真正存着这个 vault 的 MacBook 本地**的 Claude Code 里运行 ——
> 云端 / cowork 会话碰不到你电脑上的笔记文件,无法完成接入。

---

```
【任务】把我的 Obsidian 笔记库一次性接入 Git 同步和 Claude,全程自动完成。
除非遇到真正无法继续的情况(下面已标明),否则不要中途反复问我,直接做完。

【请先替换成我的实际值】
- VAULT 路径:<例如 /Users/me/Documents/MyBrain>
- 仓库:<owner/name 形式,例如 myskme/brain>

【前置检查】
确认 VAULT 路径存在、且目录里含 .obsidian 子目录(说明它是 Obsidian 库)。
若不满足,停下并告诉我(可能是这台电脑上没有这个 vault)。

【自动完成下列全部步骤】
1. 创建私有 GitHub 仓库:
   - 该仓库已存在 → 直接用;
   - 否则若本机 gh CLI 可用且已登录 → 执行 gh repo create <owner/name> --private;
   - 若 gh 不可用或未登录 → 停下来让我去 github.com/new 手动建一个 Private 仓库,
     我建好后会让你继续。
2. 克隆套件:git clone https://github.com/myskme/game-design.git 到一个临时目录,
   进入后 git checkout claude/obsidian-ai-integration-FsTz6。
3. 运行套件脚本(仓库地址用 https://github.com/<owner/name>.git):
   bash <临时目录>/obsidian-second-brain/scripts/setup-vault.sh "<VAULT路径>" "<仓库地址>"
   脚本会写入 .gitignore / CLAUDE.md / .claude 配置、建目录骨架、做首次提交,
   且对已存在的文件自动跳过,不会覆盖我的笔记。
4. cd 到 VAULT,执行 git push -u origin main。
   若推送因 GitHub 认证失败,停下来告诉我如何处理。
5. 删除第 2 步的临时目录。
6. 完成后输出一份简短报告:笔记总数、收件箱待整理数,以及「我接下来要手动做的事」:
   - 在每台 MacBook 的 Obsidian 里装 Obsidian Git 插件,backup/pull 间隔设 10 分钟;
   - 第二台 MacBook 先 git clone 仓库再用 Obsidian 打开;
   - iPhone/iPad 在 Obsidian 装 Obsidian Git 插件,用 HTTPS + GitHub Token 克隆。

【硬性要求】绝不删除或覆盖我已有的任何笔记。只有遇到上面明确标注的两处
(gh 不可用、推送认证失败)才停下来问我,其余一律自动完成。
```

跑完后,vault 已在 GitHub 私有仓库里、带好 Claude 配置。剩下的只有各设备装
Obsidian Git 插件这一步 GUI 操作(Claude 会在报告里再列一遍)。详见 `README.md`。
